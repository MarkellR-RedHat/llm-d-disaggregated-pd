TL;DR for Your Architecture Review

  • Prefill is compute-bound. Decode is memory-bound. Running them on the same GPU leaves performance on the table.
  • Disaggregated architecture routes prefill to dedicated pods, transfers KV caches via RDMA in ~1ms, then decode pods generate tokens.
  • Result: 19.8x faster TTFT on cached requests, 30-60% higher throughput, and GPU utilization that doesn't yo-yo between phases.
  • It's not magic. It's just specialization.
If you're running >10 concurrent requests on a 70B+ model with contexts over 4K tokens, disaggregation pays for itself on day one.

The Routing Trap

Intelligent routing is essential to realize the full benefit of prefix caching.

Deployment Guide

        
Four YAML configs. Copy the one that matches your setup.

        


            
            

                

                    
1

                    
Standard Deployment

                

                
Start with a baseline llm-d deployment with prefix-aware routing. Pods handle both prefill and decode.

                

                    

apiVersion: inference.llm-d.ai/v1alpha1
kind: LLMInferenceService
metadata:
  name: llama-3-3-70b-fp8
  namespace: llm-d
spec:
  modelArtifact:
    uri: "oci://quay.io/llm-d/models:llama-3.3-70b-instruct-fp8-kv-int8"
  inferenceConfig:
    replicas: 2
    accelerator:
      type: nvidia.com/gpu
      count: 4
      product: H200
  routing:
    strategy: prefix-aware

                

            


            
            

                

                    
2

                    
Add Prefill Workers

                

                
Add dedicated prefill workers for the compute-intensive prompt processing phase.

                

                    

apiVersion: inference.llm-d.ai/v1alpha1
kind: LLMInferenceService
metadata:
  name: llama-3-3-70b-fp8
  namespace: llm-d
spec:
  modelArtifact:
    uri: "oci://quay.io/llm-d/models:llama-3.3-70b-instruct-fp8-kv-int8"
  inferenceConfig:
    replicas: 2
    accelerator:
      type: nvidia.com/gpu
      count: 4
      product: H200
  prefill:
    replicas: 1
    accelerator:
      type: nvidia.com/gpu
      count: 4
      product: H200
  routing:
    strategy: prefix-aware

                

            


            
            

                

                    
3

                    
Enable KV Cache Transfer

                

                
Configure high-speed KV cache transfer using NIXL with RDMA transport for minimal latency between pods.

                

                    

apiVersion: inference.llm-d.ai/v1alpha1
kind: LLMInferenceService
metadata:
  name: llama-3-3-70b-fp8
  namespace: llm-d
spec:
  modelArtifact:
    uri: "oci://quay.io/llm-d/models:llama-3.3-70b-instruct-fp8-kv-int8"
  inferenceConfig:
    replicas: 2
    accelerator:
      type: nvidia.com/gpu
      count: 4
      product: H200
  prefill:
    replicas: 1
    accelerator:
      type: nvidia.com/gpu
      count: 4
      product: H200
  kvCacheTransfer:
    connector: nixl
    transport: rdma
  routing:
    strategy: prefix-aware

                

            


            
            

                

                    
4

                    
Configure Disaggregated Routing

                

                
Switch to disaggregated-prefill-decode routing with KV-affinity for decode pods to maximize cache reuse.

                

                    

apiVersion: inference.llm-d.ai/v1alpha1
kind: LLMInferenceService
metadata:
  name: llama-3-3-70b-fp8
  namespace: llm-d
spec:
  modelArtifact:
    uri: "oci://quay.io/llm-d/models:llama-3.3-70b-instruct-fp8-kv-int8"
  inferenceConfig:
    replicas: 2
    accelerator:
      type: nvidia.com/gpu
      count: 4
      product: H200
  prefill:
    replicas: 1
    accelerator:
      type: nvidia.com/gpu
      count: 4
      product: H200
  kvCacheTransfer:
    connector: nixl
    transport: rdma
  routing:
    strategy: disaggregated-prefill-decode
    prefillRouting:
      loadBalancing: least-load
    decodeRouting:
      loadBalancing: kv-affinity

pd-sidecar

                    
Runs alongside each vLLM engine pod. Coordinates KV cache metadata exchange, registers memory buffers for RDMA, and manages the lifecycle of cache transfers between prefill and decode pods.

                

                

                    
NIXL (NVIDIA Inference Xfer Library)

                    
A high-performance data transfer library that enables zero-copy GPU memory transfers between pods using RDMA. Provides the low-level transport layer for moving KV cache blocks at hardware bandwidth speeds.

                

                

                    
RDMA Transport

                    
Remote Direct Memory Access enables GPU-to-GPU transfers that bypass the CPU and OS kernel entirely. Combined with NIXL, this delivers sub-millisecond KV cache transfers even for large context windows.

                

                

                    
Request Flow

                    
1. Router receives request, selects prefill pod via least-load balancing. 2. Prefill pod processes prompt, generates KV cache. 3. KV cache transfers to decode pod via NIXL/RDMA. 4. Decode pod generates tokens using cached attention states.
