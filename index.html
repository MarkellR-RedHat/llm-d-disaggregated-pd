Why Separate Prefill and Decode?

Prefill

Processing 1,000 input tokens simultaneously

GPU Compute 95%
Memory Bandwidth 55%
Compute-bound — GPU cores fully utilized
Decode

Generating tokens one at a time

GPU Compute 15%
Memory Bandwidth 90%
Memory-bound — waiting on KV cache reads

Key Insight

Mixing both phases on the same GPU leads to interference — prefill starves decode of memory bandwidth, while decode leaves GPU compute underutilized during generation. By separating these workloads, each pod type can be optimized for its specific resource profile.

Real Benchmark Data

Key Insight

At 32K tokens, cached TTFT is 19.8x faster than cold — dropping from 4.74s to 0.24s. This dramatic improvement shows the power of prefix caching when combined with intelligent routing.

Key Insight

Near-linear scaling: 50x concurrency yields 32.6x throughput increase. Disaggregated architecture maintains efficiency even under heavy load by preventing phase interference.

Key Insight

Without intelligent routing, requests hit random pods — cache reuse drops to near zero (0.6-1.0x), eliminating the primary performance benefit. Cache-aware routing is essential for realizing the gains from prefix caching.

Deployment Guide

1

Standard Deployment

Start with a basic llm-d deployment with prefix-aware routing. This baseline configuration uses pods that handle both prefill and decode. 

apiVersion: inference.llm-d.ai/v1alpha1
kind: LLMInferenceService
metadata:
  name: llama-3-3-70b-fp8
  namespace: llm-d
spec:
  modelArtifact:
    uri: "oci://quay.io/llm-d/models:llama-3.3-70b-instruct-fp8-kv-int8"
  inferenceConfig:
    replicas: 2
    accelerator:
      type: nvidia.com/gpu
      count: 4
      product: H200
  routing:
    strategy: prefix-aware
2

Add Prefill Workers

Add dedicated prefill workers to handle prompt processing. These pods will focus exclusively on the compute-intensive prefill phase. 

apiVersion: inference.llm-d.ai/v1alpha1
kind: LLMInferenceService
metadata:
  name: llama-3-3-70b-fp8
  namespace: llm-d
spec:
  modelArtifact:
    uri: "oci://quay.io/llm-d/models:llama-3.3-70b-instruct-fp8-kv-int8"
  inferenceConfig:
    replicas: 2
    accelerator:
      type: nvidia.com/gpu
      count: 4
      product: H200
  prefill:
    replicas: 1
    accelerator:
      type: nvidia.com/gpu
      count: 4
      product: H200
  routing:
    strategy: prefix-aware
3

Enable KV Cache Transfer

Configure high-speed KV cache transfer between prefill and decode pods using NixL with RDMA transport for minimal latency. 

apiVersion: inference.llm-d.ai/v1alpha1
kind: LLMInferenceService
metadata:
  name: llama-3-3-70b-fp8
  namespace: llm-d
spec:
  modelArtifact:
    uri: "oci://quay.io/llm-d/models:llama-3.3-70b-instruct-fp8-kv-int8"
  inferenceConfig:
    replicas: 2
    accelerator:
      type: nvidia.com/gpu
      count: 4
      product: H200
  prefill:
    replicas: 1
    accelerator:
      type: nvidia.com/gpu
      count: 4
      product: H200
  kvCacheTransfer:
    connector: nixl
    transport: rdma
  routing:
    strategy: prefix-aware
4

Configure Routing

Switch to disaggregated-prefill-decode routing strategy with KV-affinity for decode pods to maximize cache reuse. 

apiVersion: inference.llm-d.ai/v1alpha1
kind: LLMInferenceService
metadata:
  name: llama-3-3-70b-fp8
  namespace: llm-d
spec:
  modelArtifact:
    uri: "oci://quay.io/llm-d/models:llama-3.3-70b-instruct-fp8-kv-int8"
  inferenceConfig:
    replicas: 2
    accelerator:
      type: nvidia.com/gpu
      count: 4
      product: H200
  prefill:
    replicas: 1
    accelerator:
      type: nvidia.com/gpu
      count: 4
      product: H200
  kvCacheTransfer:
    connector: nixl
    transport: rdma
  routing:
    strategy: disaggregated-prefill-decode
    prefillRouting:
      loadBalancing: least-load
    decodeRouting:
      loadBalancing: kv-affinity
