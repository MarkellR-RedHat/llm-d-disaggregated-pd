Benchmark Data

Real performance data from Llama 3.3 70B Instruct FP8 on NVIDIA H200 GPUs.

Key Insight

At 32K tokens, cached TTFT is 19.8x faster than cold -- dropping from 4.74s to 0.24s. Prefix caching combined with intelligent routing delivers transformative latency improvements at scale.

Key Insight

Near-linear scaling: 50x concurrency yields a 32.6x throughput increase. Disaggregated architecture maintains efficiency under heavy load by preventing phase interference.

Key Insight

Without cache-aware routing, requests scatter across pods and cache hit rate drops to near zero. Intelligent routing is essential to realize the full benefit of prefix caching.

pd-sidecar

Runs alongside each vLLM engine pod. Coordinates KV cache metadata exchange, registers memory buffers for RDMA, and manages the lifecycle of cache transfers between prefill and decode pods.

NIXL (NVIDIA Inference Xfer Library)

A high-performance data transfer library that enables zero-copy GPU memory transfers between pods using RDMA. Provides the low-level transport layer for moving KV cache blocks at hardware bandwidth speeds.

RDMA Transport

Remote Direct Memory Access enables GPU-to-GPU transfers that bypass the CPU and OS kernel entirely. Combined with NIXL, this delivers sub-millisecond KV cache transfers even for large context windows.

Request Flow

1. Router receives request, selects prefill pod via least-load balancing. 2. Prefill pod processes prompt, generates KV cache. 3. KV cache transfers to decode pod via NIXL/RDMA. 4. Decode pod generates tokens using cached attention states.

Deployment Guide

Progressive configuration from standard to fully disaggregated, using the LLMInferenceService CRD.

1

Standard Deployment

Start with a baseline llm-d deployment with prefix-aware routing. Pods handle both prefill and decode.

apiVersion: inference.llm-d.ai/v1alpha1
kind: LLMInferenceService
metadata:
  name: llama-3-3-70b-fp8
  namespace: llm-d
spec:
  modelArtifact:
    uri: "oci://quay.io/llm-d/models:llama-3.3-70b-instruct-fp8-kv-int8"
  inferenceConfig:
    replicas: 2
    accelerator:
      type: nvidia.com/gpu
      count: 4
      product: H200
  routing:
    strategy: prefix-aware
2

Add Prefill Workers

Add dedicated prefill workers for the compute-intensive prompt processing phase.

apiVersion: inference.llm-d.ai/v1alpha1
kind: LLMInferenceService
metadata:
  name: llama-3-3-70b-fp8
  namespace: llm-d
spec:
  modelArtifact:
    uri: "oci://quay.io/llm-d/models:llama-3.3-70b-instruct-fp8-kv-int8"
  inferenceConfig:
    replicas: 2
    accelerator:
      type: nvidia.com/gpu
      count: 4
      product: H200
  prefill:
    replicas: 1
    accelerator:
      type: nvidia.com/gpu
      count: 4
      product: H200
  routing:
    strategy: prefix-aware
3

Enable KV Cache Transfer

Configure high-speed KV cache transfer using NIXL with RDMA transport for minimal latency between pods.

apiVersion: inference.llm-d.ai/v1alpha1
kind: LLMInferenceService
metadata:
  name: llama-3-3-70b-fp8
  namespace: llm-d
spec:
  modelArtifact:
    uri: "oci://quay.io/llm-d/models:llama-3.3-70b-instruct-fp8-kv-int8"
  inferenceConfig:
    replicas: 2
    accelerator:
      type: nvidia.com/gpu
      count: 4
      product: H200
  prefill:
    replicas: 1
    accelerator:
      type: nvidia.com/gpu
      count: 4
      product: H200
  kvCacheTransfer:
    connector: nixl
    transport: rdma
  routing:
    strategy: prefix-aware
4

Configure Disaggregated Routing

Switch to disaggregated-prefill-decode routing with KV-affinity for decode pods to maximize cache reuse.

apiVersion: inference.llm-d.ai/v1alpha1
kind: LLMInferenceService
metadata:
  name: llama-3-3-70b-fp8
  namespace: llm-d
spec:
  modelArtifact:
    uri: "oci://quay.io/llm-d/models:llama-3.3-70b-instruct-fp8-kv-int8"
  inferenceConfig:
    replicas: 2
    accelerator:
      type: nvidia.com/gpu
      count: 4
      product: H200
  prefill:
    replicas: 1
    accelerator:
      type: nvidia.com/gpu
      count: 4
      product: H200
  kvCacheTransfer:
    connector: nixl
    transport: rdma
  routing:
    strategy: disaggregated-prefill-decode
    prefillRouting:
      loadBalancing: least-load
    decodeRouting:
      loadBalancing: kv-affinity
